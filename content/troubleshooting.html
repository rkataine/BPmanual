BasePlayer manual

Troubleshooting

 

Problem

Possible reason

Solution

1

Installer does not start

The installer file may not have the necessary file permissions

Set the execute permission to the installer file and run again.

Alternatively, download the BasePlayer.zip, unzip it and use Launcher.jar to start the program. JAR launcher is required for Linux systems. In macOS, click the launcher file while pressing the "control" key.

2

BasePlayer does not start

Operating system does not have Java Runtime Environment installed

Download and install Java Runtime Environment from the http://www.oracle.com (a 64-bit version is recommended).

3

BasePlayer does not start

The launcher file does not have the execute permission

Set the execute permission to the launcher file and run again.

If Launcher.jar file is used and does not work, start the program using the command line: java -jar Launcher.jar

4

BasePlayer does not start

Installed Java Runtime Environment is incompatible with the requested memory allocation

Download and install a 64-bit version of Java Runtime Environment to allocate more than 1.2 GB of memory to BasePlayer. We recommend 2 GB for larger projects.

5

BasePlayer does not start

macOS Gatekeeper does not open applications from "unidentified developers"

The instructions for this issue can be found at

https://www.imore.com/how-open-apps-unidentified-developers-mac

6

Launcher.jar does not launch the software

macOS may have Java compatibility issues

Launch program with BasePlayer.jar or use the macOS installer and launch with BasePlayer launcher. Note: memory allocation can not be increased when the BasePlayer.jar is used to launch the software

7

A genome file cannot be downloaded

Ensembl server is unavailable

Test the server in the web browser by entering ftp://ftp.ensembl.org/ to the address bar. If the server is unavailable, try downloading the genome later.

8

A genome file is corrupted or does not work

Downloading or processing of genome files may have been interrupted

Delete the genome file by clicking "File > Genomes > Add new genome", selecting the invalid genome and clicking "Remove". Download the genome again.

The genome can also be removed manually by deleting the genome folder in the Genome Directory.

9

VCF file does not open

No read permissions for the VCF or index file

Set the file read permissions on the VCF and the index file.

10

VCF file does not open

VCF file is not compressed with bgzip or is not sorted

In case an uncompressed, unindexed VCF is opened, BasePlayer will try to create an index before opening the file. An index file will not be created if the VCF is unsorted or there are no writing permissions for the index file in the folder. Sort the VCF file with the unix command "sort -k1,1V -k2,2n sample.vcf >sample_sorted.vcf". bgzip file with the unix command "bgzip sample_sorted.vcf". Give the VCF containing folder writing permissions and open the file again.

11

VCF file does not open

VCF and/or index file is corrupted

Check the VCF file manually (in Linux, use zless or tabix commands). If necessary, regenerate the VCF and/or index file.

12

Variants do not show on the sample track

Default filtering thresholds may be too high

Go to the Variant Manager and set "Min. coverage" and "Min. allelic fraction" to zero. If no variants show up despite this, check that there are variants in the selected chromosome and that the VCF/index files are not corrupt.

13

BAM file is not recognized

File naming may be incorrect

Make sure the BAM, VCF and index files share a file name prefix, e.g., sample.bam, sample.bam.bai, sample.vcf.gz, sample.vcf.gz.tbi, and rename the files if necessary.

14

BAM file does not open or reads are not shown

Problem may be the same as with VCF files

See the VCF file troubleshooting steps reported above. If necessary, check the BAM file manually using the "samtools" command.

15

BED file does not open or regions are not shown

Problem may be the same as with VCF files

See VCF file troubleshooting steps reported above. If necessary, check the BED file using any text editor.

16

Somatic variants are not shown when opening a VCF containing somatic variants

Somatic variants are not found in the last column of the VCF file

Somatic variants should be reported in the last column of the VCF. Remove all other columns except for the one containing somatic variant genotypes.

17

Cannot write output file

No write permissions to the folder

Set write permissions to the target folder.

18

Output file is empty

No variants passed the filters and analysis parameters

If no variants show up in the variant manager, loosen the analysis criteria and try to annotate the variants again.

19

Annotation fails when using a CADD track file. Error message: "Memory allocation exceeded in Annotating variants with <filename>"

Memory consumption exceeds the maximum allocated memory

Increase allocated memory for BasePlayer. We recommend using at least 2 GB when using a CADD annotation. Alternatively, go to "Tools > Settings > General" and change "Processing window size (bp)" to 100,000 and press the "Enter" key.

20

CADD track file does not work

Correct annotation columns are not selected

See Supplementary Tutorial for instructions

21

Can not right click or scroll screen vertically with the right mouse button

The mouse has only one button

 

In some Mac systems, the mouse does not have a right button. Press the "Ctrl" key while clicking with the mouse. More instructions at https://www.wikihow.com/Right-Click-on-a-Mac
